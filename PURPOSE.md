Ключевая цель - сделать возможность подключаться к одной ноде с нескольких устройств, имитируя подключение нескольких виртуальных нод

Три режима:
    DEFAULT
    SHARED_NODE
    ACCESS_NODE

DEFAULT
    Всё как в оригинале
    Mesh - Node - User

SHARED_NODE
    При первом запуске устанавливается подключение к ноде стандартным способом (RANDOM_PIN) - этот пользователь будет админом ноды.
    После успешного подключения остальные пользователи смогут подключиться через GUEST_PIN (fixed pin, определённый отдельно)
    LATER TODO: admin can manage guest accounts
    Приложение пользователя переписывать не надо, каждое приложение будет думать что подключается к собственной ноде
    Настройки соедниения может менять только админ, у гостей будет silent fail/explicit fail
    Гости могут только указывать user-данные своей (виртуальной) ноды
    Кастомная прошивка будет заниматься менеджментом пакетов между физической нодой и виртуальными

    Как оно будет работать?
    Как якобы множество псевдо-нод подключенной к одной визической.

    Mesh - Physical Node (router) - Virtual Pseudo-node (user #1) - User #1

ACCESS_NODE
    Похожая идея, но предпологается, что пользователи не пользуются стандартными приложениями, а используют веб-интерефейс (получают доступ по сети/подключаясь к сайту, который хостится на устройстве, подкюченному к ноде по Bluetooth), нода также маршрутизирует множество виртуальных нод, но к ней самой нет множества подключений, этим занимается сервер

    Mesh - Physical Node (router) - Server - User #1
